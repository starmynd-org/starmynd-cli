# @starmynd1/cli

StarMynd CLI, manage your workspace from the terminal.

## What is it

The StarMynd CLI is the terminal counterpart to the StarMynd web app. It lets you work with the same workspace, the same knowledge bases, and the same entities (agents, workflows, skills, rules) without leaving your shell.

Use it to scaffold a local `.starmynd/` directory against a workspace, pull remote state into version-controlled YAML, edit entities in your editor, validate them against the StarMynd schemas, and push them back. For knowledge bases you can export a namespace to disk, bulk-upload a folder of markdown or HTML, preview HTML nodes in the browser, and diff local files against remote.

It talks to the same API as the web app. Authentication is per-workspace; switching workspaces is a single command.

## Install

```bash
npm i -g @starmynd1/cli
```

Node 18 or later is required (the CLI uses native `fetch` and the modern `AbortSignal.timeout` API).

Verify:

```bash
starmynd --version
```

## Authenticate

```bash
starmynd auth login
```

This starts a local callback server on port `19275` and opens your browser to the StarMynd authorize page. After you approve, the browser shows a success screen and credentials are written to your home directory.

If the browser does not open, the terminal prints a URL beginning with `https://app.starmynd.com/api/cli/auth/authorize?redirect_uri=...`. Open that URL manually and the rest of the flow completes normally. The login attempt times out after two minutes.

API-key login is also supported:

```bash
starmynd auth login --api-key <key>
```

Check your current session at any time:

```bash
starmynd auth status
```

## Quick start

Start from an empty directory, connect it to a workspace, pull state, and validate:

```bash
mkdir my-workspace && cd my-workspace
starmynd auth login
starmynd init --slug <your-workspace-slug>
starmynd validate .starmynd/
```

`init` writes `.starmynd/config.yaml`, `GUIDE.md`, and the entity folders (`agents/`, `workflows/`, `skills/`, `knowledges/`, `namespaces/`, `governance/`). From there, edit a YAML file and run `starmynd push .starmynd/agents/my-agent.yaml` to send the change back.

For a ready-made starter you can fork and customize, see [starmynd-org/starmynd-starter](https://github.com/starmynd-org/starmynd-starter).

## Commands

All commands below are registered in `src/index.ts`. Run `starmynd <command> --help` for the full flag list.

### auth

Manage credentials and the active workspace.

| Subcommand | Purpose |
|------------|---------|
| `auth login` | OAuth browser flow (or `--api-key <key>`). |
| `auth status` | Show email, workspace, auth type, and token expiry. |
| `auth switch-workspace [slug]` | Switch workspaces, or use `--list` to see your options. |
| `auth logout` | Clear stored credentials. |

```bash
starmynd auth login
starmynd auth switch-workspace --list
starmynd auth switch-workspace acme
```

### init

Initialize a `.starmynd/` workspace connection in the current directory.

```bash
starmynd init --slug acme
```

Flags: `--slug`, `--api-key`, `--endpoint` (defaults to `https://app.starmynd.com`), `--force` (overwrite an existing `.starmynd/`). If called with no `--slug`, `init` prompts interactively. It refuses to overwrite an existing config unless `--force` is passed, and fails fast with a clear message if you are not authenticated.

### status

Show workspace connection, auth validity, last-pull freshness, and remote entity counts.

```bash
starmynd status
```

### pull

Pull workspace state into the local `.starmynd/` directory. Writes one YAML file per entity under `agents/`, `workflows/`, `knowledges/`, `skills/`, and a namespaces index under `namespaces/_index.yaml`. Regenerates `GUIDE.md`.

```bash
starmynd pull
starmynd pull --only agents,workflows
```

### push

Push local entity files to StarMynd. Accepts either a single file or a directory. Runs client-side validation first and refuses to push if anything fails.

```bash
starmynd push .starmynd/agents/my-agent.yaml
starmynd push .starmynd/agents/ --dry-run
```

### diff

Compare local entity hashes under `.starmynd/` against the remote. Reports new-locally, modified, and remote-only entities. Skips internal files (`config.yaml`, `namespaces/`, `governance/`).

```bash
starmynd diff
```

### validate

Validate local YAML or markdown entity files against the StarMynd schemas. Accepts a file or a directory (defaults to `.`). Skips scaffold files generated by `init` (`GUIDE.md`, `config.yaml`, `governance/config.yaml`, `namespaces/_index.yaml`) so a fresh `init` validates clean.

```bash
starmynd validate
starmynd validate .starmynd/agents/
starmynd validate .starmynd/ --remote
```

`--remote` sends the same entities to the API for a stricter server-side check.

### create

Create a single entity from a YAML or markdown file.

```bash
starmynd create agent --from ./new-agent.yaml
```

Valid types: `agent`, `workflow`, `knowledge`, `skill`, `rule`.

### list

List entities of a given type, or enumerate knowledge namespaces.

```bash
starmynd list agents
starmynd list workflows --status active
starmynd list knowledge
```

Valid types: `agents`, `workflows`, `knowledge`, `skills`, `rules`, `catalogs`. Filter by `--status active|draft|archived`.

### get

Fetch a single entity by type and slug, print as YAML or JSON.

```bash
starmynd get agent briefing-agent
starmynd get workflow daily-report --format json
```

### kb

Knowledge base operations scoped to a namespace.

| Subcommand | Purpose |
|------------|---------|
| `kb list --namespace <id\|slug>` | List all nodes in a namespace. |
| `kb upload --namespace <id> --file <path>` | Upload one file as a node. |
| `kb upload-dir --namespace <id> --directory <dir>` | Bulk upload `.md`, `.html`, `.htm`, `.txt` files (batched 100 at a time). |
| `kb pull --namespace <id\|slug> [--output <dir>]` | Export a namespace to a zip and extract it (defaults to `./kb-export`). |
| `kb diff --namespace <id> --directory <dir>` | Compare a local KB folder against the remote namespace. |
| `kb preview --file <path> [--responsive]` | Render an HTML KB file in the browser. |

```bash
starmynd kb list --namespace public-docs
starmynd kb upload-dir --namespace public-docs --directory ./articles
starmynd kb pull --namespace public-docs --output ./kb
starmynd kb preview --file ./article.html --responsive
```

`kb pull` uses the `unzip` binary if available; otherwise the downloaded zip is kept in place.

### preview

Preview an arbitrary HTML component file in the browser (same rendering as `kb preview`, but accepts any local HTML file).

```bash
starmynd preview ./component.html
starmynd preview ./component.html --responsive
```
